监管与新发市场监控日报

每日八点自动出监管日报

TL;DR

每个工作日早 8 点，自动抓取证监会、基金业协会的当日政策动态，以及全市场新发基金数据，调用大模型筛选并生成日报，约 40 秒送达邮箱。已稳定运行，做产品判断从「人工盯梢」变成「邮件就位」。

痛点 · PROBLEM

监管信息分散是产品岗信息流的天然痛点：证监会、基金业协会的政策更新各自散落在多个官网栏目，新发基金市场又要去 Wind 或同花顺单独跑数据。要做到「不遗漏」，传统流程依赖经验抽查或定期手动巡检，结构上不够稳。

想要的是一个系统化的信息底——早上一打开邮箱，今天的监管要闻与新发市场就摆在那里，让产品判断的输入端从「靠盯」变成「邮件就位」。

怎么做的 · HOW IT WORKS

工具是一个 Python 后台作业，由 macOS launchd 在每个工作日早 8 点唤起。流程：① 多源爬虫并发抓证监会、基金业协会的当日栏目；② 用 akshare 抓全市场新发基金数据，按关注关键词过滤；③ 调用大模型对原始条目做相关性筛选与摘要；④ 渲染成 HTML 和 Markdown 日报；⑤ 通过 SMTP 推送到邮箱。

做得最较真的两个工程细节：稳定性——加了「等网络就绪（最多 3 分钟）+ akshare 接口重试」两道保护，应对 Mac 唤醒后 WiFi 还没连上的情况；邮件 SMTP 加超时和重试，应对 QQ/163 临时不响应；合规友好——只抓公开列表页的标题与链接、请求间隔 3-5 秒、不绕过任何反爬，单源失败就标「请手动查看」而不是隐藏。

$ python3 main.py
→ output/daily_20260617.md + .html，邮件已发送（约 40 秒）
真实输出 · OUTPUT
邮件内日报 · 监管动态 + 新发市场表格
成果 · OUTCOME

已稳定运行，每个工作日早 8:01 准时把日报推到邮箱，从「人工盯梢半小时」变成「邮件就位」。

确保监管要闻与新发基金市场动态不遗漏，产品判断的输入端从此自动化。

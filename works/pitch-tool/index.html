一页通与路演材料生成器

发行材料一键生成与合规拦截

TL;DR

输入一只新基金的产品要素 YAML，一条命令产出 A4 一页通 HTML、十二页路演 PPT 框架、渠道 FAQ 与合规自检报告四件套；内置红线词校验和风险提示文案逐字比对，不合规直接拦截。

痛点 · PROBLEM

每发行一只新基金，渠道部门要的标配是一页通 + 路演 PPT + 渠道 FAQ三件套，再加产品部自己要做的合规自检。每一份的版式、字段、合规话术都大同小异，但都要一字一字重做。

更要命的是合规——风险提示文案在 PPT、一页通、FAQ 里必须逐字一致，多一个字、少一个字都不行；行业还有一长串「红线词」（如「保本」「稳赚」「高收益」等）不能出现。人工核对一遍要一小时，错一个字就要返工。

想要一个东西，录入产品要素表，机器把四件套全生成、合规自检全做完，不合规直接拦截，让人只做判断、不做体力。

怎么做的 · HOW IT WORKS

工具是一个 Python 命令行程序。流程：① 读取 product.yaml 产品要素表；② 调用大模型生成亮点/策略/背景/FAQ 文案；③ 渲染 A4 一页通 HTML（浏览器 Cmd+P 直接存 PDF）；④ 生成十二页路演 PPT 框架，红字标【待补充】；⑤ 跑合规自检——红线词扫描 + 风险提示文案逐字比对，输出合规报告。

工程上做得比较细的几处：模板与逻辑分离，想改一页通版式只改 HTML 模板，不动 Python；降级机制，大模型调用失败（断网/key 失效）不让流程崩，自动改用占位稿；合规拦截，红线词命中则程序以失败状态退出，材料不可流出，强制改词重跑；严格不上线真实产品，使用手册顶部标了红字警示，测试一律虚构产品。

$ python3 main.py --input product.yaml
→ output/产品代码_日期/ · 一页通.html + 路演框架.pptx + 渠道FAQ.md + compliance_report.md
真实输出 · OUTPUT
一页通.html · A4 版式，Cmd+P 直接出 PDF
compliance_report.md · 合规自检报告
成果 · OUTCOME

渠道材料制作 + 合规检查从「半天人工」变成「几分钟自动」，并内置红线拦截把不合规挡在源头。

降低重复劳动，把发行准备的精力释放出来用于判断与协同。

01 / fund_tool

同类基金竞品分析器

一条命令出竞品分析报告
痛点

产品岗做竞品分析时，需手工查询数据、拼接表格，重复劳动密集且耗时较长。

成果

实现竞品分析全流程自动化，输出 Markdown 报告与 Excel 对比表，替代原有手工查询与制表环节。

怎么做的

输入基金代码即自动抓取同类竞品数据，横向比较业绩与风险指标，调用大模型生成 Markdown 分析报告与 Excel 对比表，一条命令端到端完成。

$ python3 main.py --code 007994
→ output/007994_20260612/report.md + comparison.xlsx
Pythonakshare 数据接口大模型分析Markdown 报告Excel 对比表
02 / reg_daily

监管与新发市场监控日报

每日八点自动出监管日报
痛点

监管政策分散在多个官网，人工盯梢易遗漏，汇总整理耗时长，难以保证时效。

成果

已稳定运行，工作日早八点约四十秒自动生成并推送日报，确保监管与新发市场信息不遗漏。

怎么做的

每个工作日早八点自动抓取证监会与基金业协会政策动态，以及全市场新发基金数据，调用大模型提炼摘要并生成日报，自动邮件推送，约四十秒出报告。

$ python3 main.py --days 1
→ 约 40 秒生成日报，工作日 08:00 自动送达邮箱
Python多源爬虫大模型摘要邮件推送launchd 定时
03 / pitch_tool

一页通与路演材料生成器

发行材料一键生成与合规拦截
痛点

新基金发行渠道材料制作重复劳动密集，合规检查依赖人工逐字核对，效率与准确率受限。

成果

将渠道材料制作与合规检查自动化，输出四件套并内置红线拦截，减少人工重复核对。

怎么做的

输入产品要素 YAML，一条命令产出 A4 一页通 HTML、十二页路演 PPT、渠道 FAQ 及合规自检报告四件套；内置红线词校验与风险提示逐字比对，不合规即拦截。

$ python3 main.py product.yaml
→ 一页通.html + 路演框架.pptx + 渠道FAQ.md + 合规报告
PythonYAML 要素输入HTML / PPTX 模板红线词校验合规自检
